# FastSpec
Spec-driven development (SDD) for AI agents and other machines

Key concepts: 
- Made for agents, interface for humans
- Fast and efficient, enviromnent friendly
- Decentralized in mind
- As complex, as needed to solve tasks. Lightweight is priority not the goal.
- Formal logic involved

Aim: write software that will help to write more complex software with power of machine learning and computer science systems.

Technical stack:
- Programming language: mostly Rust. It is efficient and dummy proven. Excelent scalebility and supporting other good rustlang software. Interfaces for humans and for agents.
- Specs format: YAML files. Dont waste tokens for different programming languages (remove this from specs) and tokens saving format (but human readable)
- MCPs with logical tests for modules: memory (inspired by basic-memory mcp and git hierarchy), spec designer (inspired by contex7, remove unnessasary information), swarm agents manager (inspired by https://github.com/eyaltoledano/claude-task-master and computer science consensus algorithms), knowledge base (inspired by RAGs and coding RAGs, with 2D rags and 3D rags) 
- Use mermaid for diagrams, MDX if needed. Other comfortable formats from leading OSS.
- Python for ML related experiences. Jax stack for custom research models.
